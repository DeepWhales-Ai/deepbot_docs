<H2 id="rules">The rules you set</H2>
<KeyValueGrid items={[
  { k: "Budget", v: "Total capital Dale is allowed to deploy. Hard cap. Once it is hit, Dale stops opening new positions." },
  { k: "Per trade cap", v: "Maximum size on any single trade. The same rule you'd give a junior trader at a desk." },
  { k: "Safety floor", v: "Minimum safety score Dale will accept before entering. Honeypots, blacklists, sell tax flips, deployer history. Anything below the floor is skipped." },
  { k: "Chains", v: "Which chains Dale is allowed to trade on. Limit to the ones you trust." },
  { k: "Style match", v: "Dale only enters trades that match your historical pattern. Out of style means out of consideration." },
  { k: "Kill switch", v: "One sentence to stop everything. \"Dale, stop autonomous.\" All open positions remain. New entries cease." },
  { k: "Time limit", v: "How long autonomous mode runs before it expires automatically. Default twelve hours. You can extend in chat." },
]} />

<Callout kind="info" title="Style match is the difference">
  A bot that scans for any signal will fire constantly and lose. Dale only acts on signals
  that align with the kind of trade you have historically taken. The same memory layer that
  warns you when you're off pattern is what governs autonomous mode.
</Callout>

<H2 id="why">Why this is the difference</H2>
<Para>
  Other products call this "copy trading" or "auto buy" or "bot mode" and mean a hardcoded
  rule that fires on a single trigger. That is a script, not a partner. It will fire on
  garbage and it will fire while you sleep, exactly the wrong combination.
</Para>
<Para>
  Dale's autonomous mode is the same model that watches your patterns, calibrated by the
  same memory, run inside the same safety stack, fed by the same intelligence layer that
  powers the rest of the platform. It is the product, not a feature.
</Para>
